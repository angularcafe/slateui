components.json

The components.json file is the configuration file for SlateUI. It allows you to customize the behavior of the CLI, set your preferred styling approach, and configure project-specific settings.

Overview

The components.json file is automatically created when you first run the SlateUI CLI. It stores your preferences and project configuration, making it easier to add components consistently across your project.

Pro tip

You can manually edit this file to change your preferences, or regenerate it by running slateui init again.

File Location

The components.json file is typically placed in your project root directory: 

my-angular-app/
├── components.json
├── package.json
├── angular.json
├── tailwind.config.js
└── src/
    └── app/
        └── components/

Configuration Schema

The configuration file supports the following main sections:

Basic Configuration

  • style - Choose between "default" or "new-york" styling
  • framework - Set to "angular" for Angular projects
  • $schema - Optional schema reference for IntelliSense

Tailwind Configuration

  • config - Path to your tailwind.config.js file
  • css - Path to your global CSS file
  • baseColor - Primary color for the design system
  • cssVariables - Enable/disable CSS custom properties

Directory Aliases

  • components - Where components will be generated
  • utils - Where utility functions will be placed

TypeScript Configuration

  • strict - Enable strict TypeScript checking
  • strictNullChecks - Enable strict null checking

Import Aliases

  • components - Import alias for components
  • utils - Import alias for utilities

Style Options

Choose your preferred styling approach:

Default Style

The default style provides a clean, modern design with subtle shadows and smooth transitions. Use "style": "default" in your configuration.

New York Style

A more vibrant style with stronger shadows and bolder visual elements. Use "style": "new-york" in your configuration.

Tailwind Configuration

Configure how Tailwind CSS is set up in your project:

Config File

Specify the path to your Tailwind configuration file using "config": "tailwind.config.js".

CSS File

Specify the path to your global CSS file where Tailwind directives are imported using "css": "src/styles.css".

Base Color

Choose your preferred base color for the design system using "baseColor": "slate".

Available colors include: slate, gray, zinc, neutral, stone, red, orange, amber, yellow, lime, green, emerald, teal, cyan, sky, blue, indigo, violet, purple, fuchsia, pink, rose

CSS Variables

Enable or disable CSS custom properties for theming using "cssVariables": true.

Framework Configuration

Specify your target framework and its configuration:

Use "framework": "angular" for Angular projects.

Currently supported frameworks:

  • angular - Angular 20+ with standalone components
  • react - React 18+ with TypeScript
  • vue - Vue 3+ with Composition API
  • svelte - Svelte 5+ with TypeScript

Directory Aliases

Configure where components and utilities are placed in your project:

Components Directory

The directory where SlateUI components will be generated. Use "components": "src/app/components". This should match your project structure.

Utils Directory

The directory where utility functions (like cn for class merging) will be placed. Use "utils": "src/lib/utils".

TypeScript Configuration

Configure TypeScript settings for generated components:

Strict Mode

Enable strict TypeScript checking for better type safety using "strict": true.

Strict Null Checks

Enable strict null checking to prevent null/undefined errors using "strictNullChecks": true.

Import Aliases

Configure import aliases for cleaner import statements:

Use "components": "@/components" and "utils": "@/lib/utils".

These aliases will be used in generated components and can be configured in your TypeScript configuration: 

// tsconfig.json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"]
    }
  }
}

CLI Integration

The CLI automatically reads your components.json configuration when adding components: 

$ slateui add button
✔ Reading components.json...
✔ Using configuration from components.json
✔ Adding button component...
✔ Component added successfully!

You can also override specific settings during CLI execution: 

$ slateui add button --style new-york --base-color blue
✔ Overriding style: new-york
✔ Overriding base color: blue
✔ Adding button component...

Troubleshooting

Common Issues

Invalid JSON

Ensure your JSON is valid. Use a JSON validator or your editor's built-in validation.

Missing Schema

The $schema field is optional but provides IntelliSense in supported editors.

Path Issues

Ensure all file paths in your configuration exist and are accessible from your project root.
